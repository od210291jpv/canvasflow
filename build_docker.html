CanvasFlow Docker Build Instructions

This document contains the steps required to build and run the CanvasFlow services using Docker.

IMPORTANT: All commands must be executed from the project root directory:
C:\test\requirements\canvasflow

1 Build the Images

Run these commands to build the API and Web images. We use the root directory as the build context so that dependencies between projects are resolved correctly.

# Build API Image
docker build -t canvasflow-api -f CanvasFlow.Api/Dockerfile .

# Build Web Image
docker build -t canvasflow-web -f CanvasFlow.Web/Dockerfile .

2 Run the Containers

Start the containers in detached mode with port forwarding and auto-restart enabled.

API Service

Maps local port 5000 to container port 8080.

docker run -d `
  --name canvasflow-api `
  --restart always `
  -p 5000:8080 `
  canvasflow-api

Web Service

Maps local port 5001 to container port 8080.

docker run -d `
  --name canvasflow-web `
  --restart always `
  -p 5001:8080 `
  canvasflow-web

3 Verification

You can check if the containers are running using:

docker ps

Once running, you should be able to access:

  • API (Swagger): http://localhost:5000/swagger
  • Web App: http://localhost:5001
